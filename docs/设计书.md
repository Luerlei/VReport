## 1. 项目概述

### 1.1 背景与目标

UReport2 是一款基于 Spring 架构的纯 Java 高性能报表引擎，通过**单元格迭代模型**可实现任意复杂的中国式报表，并提供基于网页的可视化报表设计器。其核心能力包括：单元格迭代展开、表达式引擎、多数据源、图表、交叉表、参数面板、条件格式、多格式导出等。

UReport2 强依赖 Java 后端（JDBC 直连数据库、Spring Bean 数据源、服务器端渲染与导出），无法在纯前端环境运行。**VReport 的目标是用纯前端技术栈（Vue3 + TypeScript）复刻 UReport2 的核心功能**，让用户在浏览器中即可完成报表的设计与预览，无需部署任何后端服务。

### 1.2 设计原则

1. **零后端依赖**：所有计算、渲染均在浏览器端完成。
2. **模型对齐 UReport**：单元格模型、迭代语义、表达式语法尽量与 UReport2 保持一致，降低用户迁移成本。
3. **数据源适配前端**：用 JSON / CSV / Excel 文件上传 替代 JDBC 直连，覆盖 UReport 的数据集场景（v0.1 不含 HTTP API）。
4. **模板可迁移**：报表模板以 JSON 描述，支持 IndexedDB 持久化与文件导入导出。
5. **渐进式实现**：核心模块优先，导出与高级特性作为扩展预留。

- 行列整选高亮改为底色填充：整行/整列选中时，被选中的单元格用半透明底色（inset box-shadow）突出，配合表头高亮，直观显示选中的行/列；单个单元格仍为描边样式。

- 数据集变量的条件格式范围：单元格为数据集变量（显式绑定或内容为单个 `${ds.field}`）时，范围一律以变量形式展示并随数据量自动适配，不再显示固定坐标；编辑既有规则也会归一化为变量范围。

### 1.6 近期修复（2026-07-01 · 第二轮）

- 导出 Excel 与预览一致（消除空白 C/D 列）：向右展开会插入全局“空隙列”，预览(HTML 表格 `<td v-if="cell">`)与 HTML 导出都会跳过 null 单元格使每行收拢，而 Excel 导出此前按“绝对网格坐标”落位，残留空白列。新增 `computeAutoFlowPlacements` 复刻预览的自动排布（跳过 null、按 rowspan/colspan 避让、落位左对齐收拢），Excel 导出改用该落位，三端（预览/HTML/Excel）一致。
- 报表边缘右键菜单被裁切：右键菜单为 `position: fixed`，靠近视口右/下缘时溢出裁切。新增 `clampMenuPosition` 在渲染后按实际尺寸把菜单夹取进视口（溢出则向左/上回收，且不小于边距），并给菜单加 `max-height: calc(100vh - 12px); overflow-y:auto` 兜底。
- 预览尾部空白列不再显示：根因是表体按 `<td v-if="cell">` 流式渲染（跳过 null 空隙列），而表头曾按绝对列索引渲染，右展开后会出现“表体收拢但表头仍到 I/J”的口径不一致。现改为表头/列宽同样采用流式列布局（与 Excel 自动流式落位同语义）：仅对“有实际值/内容的文本/公式格”与“可视组件格（图表/图片/二维码/条码）”参与列排布，右展开产生的尾部占位空壳（含样式占位、空公式）不再撑出 I/J。
- 公式单元格可再次编辑：公式编辑态下 `formulaPicking=true` 时，点击任意单元格都会插入引用，连“当前正在编辑的单元格”也会被插入坐标，导致无法正常再编辑。现对“点击当前编辑单元格”做短路，保留原内容与光标行为，不再插入自引用。

## 4. 核心模块设计

### 4.1 单元格模型（Cell Model）

UReport 的核心是**单元格**。每个单元格有名称（如 A1）、坐标、合并跨度、内容、展开方向、主格关系等属性。

#### 4.1.1 数据结构

```typescript
// 单元格类型
type CellType = 'text' | 'image' | 'chart' | 'formula' | 'qrcode' | 'barcode';
// 展开方向
type ExpandDirection = 'none' | 'down' | 'right';
// 聚合方式
type Aggregate = 'none' | 'sum' | 'avg' | 'count' | 'max' | 'min' | 'group' | 'distinct';

interface Cell {
  id: string;
  name: string;              // 显示名 A1, B2
  row: number;               // 行索引（0-based）
  col: number;               // 列索引（0-based）
  rowSpan: number;           // 行合并数
  colSpan: number;           // 列合并数
  content: string;           // 原始内容（文本或表达式）
  cellType: CellType;
  expandDirection: ExpandDirection;
  // 主格关系（控制迭代分组）
  leftMasterCell?: string;   // 左主格名，控制向下展开时的分组
  topMasterCell?: string;    // 上主格名，控制向右展开时的分组
  // 数据绑定
  dataset?: string;          // 绑定的数据集名
  fieldName?: string;        // 绑定字段名
  aggregate?: Aggregate;     // 聚合方式
  // 样式与格式
  styleId?: string;          // 引用 styles 中的样式
  numberFormat?: string;     // 数字格式 "#,##0.00"
  dateFormat?: string;       // 日期格式 "yyyy-MM-dd"
  // 条件格式
  conditionIds?: string[];   // 引用 conditionFormats
  // 图表/图片配置
  chartConfig?: ChartConfig;
  imageConfig?: ImageConfig;
  // 其他
  remark?: string;           // 备注
  printable?: boolean;       // 是否参与打印
}

补充约束：

- 当单元格内容不再是“直接绑定型”数据集变量（即不再是单个 `${ds.field}`）时，`dataset`、`fieldName`、`aggregate`、`expandDirection`、`leftMasterCell`、`topMasterCell` 必须同步清理，避免删除变量后属性面板与预览仍残留陈旧配置。
- 当内容恰好是单个 `${ds.field}` 时，设计器可反向同步 `dataset/fieldName`，便于属性面板与预览保持一致。

interface RowConfig {
  index: number;
  height: number;            // px
  hidden?: boolean;
  pageBreakBefore?: boolean; // 分页符
  repeat?: boolean;          // 重复表头
}

interface ColumnConfig {
  index: number;
  width: number;             // px
  hidden?: boolean;
  freeze?: boolean;          // 冻结列
}
```

#### 4.1.2 网格结构

设计期使用二维数组 `Cell[][]` 存储原始模板网格（合并单元格用 `rowSpan/colSpan` 标记，被合并的位置存 `null` 或引用主单元格）。

渲染期由迭代引擎生成 `RenderedCell[][]`，其行/列数可能远大于模板（因展开）。

### 4.2 迭代引擎（Expand Engine）⭐核心

UReport 区别于普通表格的关键：**单元格可绑定数据集并按方向展开**，展开过程中通过"主格"关系维持分组层级。

#### 4.2.1 展开语义

- **向下展开（expandDirection = 'down'）**：单元格绑定数据集字段后，按数据集行数纵向复制该单元格所在行，每行绑定一条数据。
- **向右展开（expandDirection = 'right'）**：按数据集行数横向复制该单元格所在列。
- **主格（Master Cell）**：当一个单元格 A1 向下展开为多行时，其右侧的 B1 默认也跟随 A1 展开并绑定同一行数据；若 B1 也设置向下展开，则 B1 在 A1 的每一行内再展开。这种父子关系通过 `leftMasterCell` / `topMasterCell` 显式声明。

#### 4.2.2 算法流程

```
1. 扫描所有单元格，构建主格关系树（MasterTree）
   - 根节点：无主格的展开单元格
   - 子节点：以父格为主格的展开单元格
2. 对每个根节点执行展开：
   expand(cell, parentContext):
     data = cell.dataset ? dataset[cell.dataset] : [{}]
     for each row in data:
       ctx = parentContext.derive(row, cell)
       复制 cell 所在行/列到结果网格
       递归 expand(子节点, ctx)
3. 处理非展开单元格：按主格上下文填充
4. 计算聚合单元格（sum/avg/count...）基于其子主格范围
5. 输出 RenderedCell[][]
```

#### 4.2.3 上下文（Context）

```typescript
interface ExpandContext {
  // 当前单元格所在迭代层的数据行
  rowData: Record<string, any>;
  datasetName: string;
  // 父级上下文链（用于表达式解析时回溯）
  parent?: ExpandContext;
  // 当前展开后在结果网格中的坐标范围
  rowRange: [number, number];
  colRange: [number, number];
}
```

表达式求值时，`${field}` 从当前上下文 `rowData` 取值；若当前层无该字段，沿 `parent` 链向上查找（实现主子报表的数据继承）。

### 4.3 表达式引擎（Expression Engine）

#### 4.3.1 语法设计（对齐 UReport）

| 语法 | 含义 | 示例 |
| --- | --- | --- |
| `${expr}` | 简单表达式（字段引用或计算） | `${name}`、`${price * count}` |
| `${ds.field}` | 数据集字段变量；在静态汇总格中可表示整列数据 | `=sum(${order.amount})`、`=avg(${score.score})` |
| `=expr` | 公式表达式（支持函数） | `=sum(A1)`、`=if(${age}>18,"成年","未成年")` |
| `A1` | 单元格引用（相对） | `=A1+B1` |
| `$A1` | 绝对列引用 | 展开后仍指向 A 列 |
| `A$1` | 绝对行引用 | 展开后仍指向第 1 行 |
| `$A$1` | 完全绝对引用 | |
| `&` | 字符串拼接 | `=${name}&"-"&${age}` |

补充语义：

- 公式编辑时，若用户选择的是“绑定了数据集字段”的单元格，设计器优先写入 `${ds.field}`，避免生成固定坐标后随数据行数变化失真。
- 单个单元格引用以及“端点均落在模板边界内”的范围引用，在运行时优先按“模板源单元格”语义解析；即便有其他数据集在上方/左侧展开导致最终渲染坐标漂移，引用仍会自动跟随目标源单元格。
- 像 `D3:D100` 这类明显用于汇总展开结果的超模板大范围，仍保留按展开后渲染坐标求值的兼容语义。

#### 4.3.2 模块结构

#### 4.3.3 内置函数（v0.1 约 40+）

分类：数学（sum/avg/count/max/min/round/abs...）、字符串（concat/substring/upper/lower/replace...）、日期（now/year/month/day/formatDate...）、逻辑（if/and/or/not/case...）、聚合（sum/avg/count/max/min/group...）、格式化（format/numberFormat...）。

### 4.4 数据源（Data Source）

#### 4.4.1 统一接口

```typescript
  // 缓存的上次取数结果（供设计期预览）
  cachedRows?: any[];
}

interface DataProvider {
  fetch(dataSet: DataSet, params: Record<string, any>): Promise<any[]>;
}
```

#### 4.4.2 四种 Provider

| Provider | 配置 | 取数方式 |
| --- | --- | --- |
| `JsonProvider` | rawJson, dataPath | 解析粘贴的 JSON，按路径取数组 |
| `CsvProvider` | rawCsv, delimiter, hasHeader | PapaParse 解析为数组 |
| `ExcelProvider` | file(上传.xlsx), sheet, headerRow | SheetJS 解析指定 sheet 为对象数组 |
| `FileProvider` | file(上传), format(json/csv/xlsx) | 读取文件内容后委托给对应解析器 |

> **说明**：v0.1 不含 HTTP API 数据源。需要连接后端数据库的场景，请先用 Excel 导出数据再上传，或等待 v0.2 的 HTTP API 支持。

## 7. 关键技术难点与方案

### 7.1 单元格迭代展开（最高难度）

**难点**：UReport 的主格关系可形成多层嵌套，展开后行/列数动态变化，且表达式需在正确上下文中求值。

**方案**：
- 设计期显式声明 `leftMasterCell/topMasterCell`，避免隐式推断的歧义。
- 展开算法分两阶段：①构建主格树；②深度优先递归展开，每个展开层维护独立 `ExpandContext`。
- 对无主格的多个根展开单元格（尤其是相邻两行 `down`、相邻两列 `right`），每次执行下一个根节点展开前，必须先将其模板坐标换算到“当前渲染网格坐标”，避免前序展开插入行/列后发生覆盖。
- 表达式求值时，字段引用沿上下文链向上查找；`${ds.field}` 在静态汇总格中可解析为整列数组；单个单元格引用与模板内范围优先按模板源单元格解析，从而自动跟随展开引起的坐标漂移。
- 提供可视化主格关系视图，降低用户配置难度。

### 7.2 表达式引擎

**难点**：需同时支持字段引用、单元格引用（含绝对/相对）、函数调用、运算符优先级，且要在迭代上下文中正确求值。

**方案**：自研递归下降解析器，生成 AST 后求值。字段引用支持沿上下文链查找，也支持 `${ds.field}` 在静态汇总格中返回数据集整列。单元格引用在求值阶段优先通过模板源单元格语义定位目标渲染单元格；对超模板大范围汇总保留按展开后网格直接扫描的兼容语义。

### 7.3 大数据量渲染性能

**难点**：数据集行数过多时，展开后网格可能数万行，DOM 渲染卡顿。

**方案**：
- 预览默认仅渲染前 500 行，提示用户分页或导出查看完整数据。
- 渲染采用虚拟滚动（vue-virtual-scroller 思路）。
- 导出时不走 DOM，直接从 `RenderedCell[][]` 流式写入（ExcelJS 支持流式）。

### 7.4 PDF 导出分页

**难点**：jsPDF + html2canvas 截图方式难以精确按纸张分页，长表格易被截断。

**方案**：
- 渲染期由 `Paginator` 按纸张大小与行高预先计算分页点，在 `RenderedCell[][]` 上标记分页边界。
- 导出时按页生成 canvas，逐页 addImage。表头行按 `page.headerRows` 在每页重复。

### 7.5 HTTP 数据源跨域

**难点**：浏览器同源策略限制直接 fetch 第三方接口。

**方案**：
- 文档明确要求目标接口开启 CORS。
- 提供可选代理配置：用户在设置中填写代理 URL 前缀，请求时改走代理。
- 不内置后端代理服务（保持纯前端）。

### 7.6 撤销/重做

**难点**：设计器需支持完整的操作历史。

**方案**：采用命令模式（Command Pattern），每个设计操作封装为可逆 Command，压入历史栈。Pinia 中维护 `undoStack/redoStack`。

### 7.7 Excel 风格设计器交互

**难点**：用户期望报表设计器的交互与 Excel 一致，包括直接输入、多选、快捷键、拖拽选区等。

**方案**：
- **键盘直接编辑**：选中单元格后按任意可打印字符即进入编辑模式并输入该字符；F2 进入编辑保留原内容；Enter 确认并下移；Tab 确认并右移。
- **拖拽多选**：鼠标按下后拖动，通过 `getCellFromPoint` 实时计算行列索引并扩展选区。
- **方向键导航**：上下左右键移动选中单元格；Shift+方向键扩展选区。
- **Delete/Backspace**：清除选区内所有单元格内容。
- **Ctrl+B/I/U**：切换加粗/斜体/下划线，支持批量应用到多选区域。
- **样式工具栏（CellToolbar）**：画布上端提供图标按钮区，包含字体/字号/加粗/斜体/下划线/字体颜色/背景色/对齐/边框/合并拆分/插入行列，所有操作支持多选批量应用。
- **Grid.applyStyleToRange / setBorderToRange**：网格层提供批量样式与边框应用方法，支持对任意矩形区域一次性设置。

### 7.8 数据源与数据集编辑

**难点**：初版仅支持新建和删除，用户修改配置需删除重建。

**方案**：DataPanel 增加编辑功能，复用新建对话框，根据 `editingDs/editingSet` 状态区分新建与更新。编辑数据源后自动清除关联数据集缓存，确保下次取数使用新配置。

### 7.9 预制报表参数示例

**方案**：两个预制模板均内置参数示例：
- 销售明细报表：`region`（下拉单选：全部/华东/华南/华北/西南）、`minAmount`（数字：最低销售额）
- 学生成绩统计表：`class`（下拉单选：全部/一班/二班）、`subject`（下拉单选：全部/语文/数学）、`passScore`（数字：及格线）

参数在预览时通过 `${param.参数名}` 在表达式中引用，触发数据集重新取数与报表重渲染。

### 7.10 公式输入联想（Excel 风格）

**难点**：用户输入公式时需要参考 Excel 的联想提示，降低记忆成本。

**方案**：
- 新增 [Autocomplete.ts](file:///d:/AI/trae/VReport/src/core/expression/Autocomplete.ts) 模块，提供 `matchSuggestions`（函数名前缀匹配）和 `getParamHint`（括号内参数签名提示）。
- CellCanvas 编辑器集成联想下拉列表：输入 `=` 开头后，键入函数名前缀自动弹出匹配列表（最多 8 项），上下键导航，Enter/点击选中后自动补全函数名和括号。
- 参数提示框：光标进入括号内时，底部显示当前函数签名和说明。
- 当用户在公式模式下点选单元格时：若目标单元格绑定了数据集字段，则自动插入 `${ds.field}`；若目标为普通单元格，则插入普通单元格引用，并在运行时按模板源单元格语义自动跟随展开后的坐标变化。

### 7.11 预览默认展示

**方案**：进入预览页时自动用参数默认值渲染报表。参数默认值留空表示"全部"，即展示所有数据行，无需手动点击查询。

### 7.12 面板可收起

**方案**：设计器左右面板支持点击收起/展开，收起后仅保留 20px 折叠条，最大化画布空间。右侧面板折叠条在左侧（order: -1），左侧面板折叠条在右侧。

### 7.13 单元格编辑提交修复

**问题**：点击其他单元格时，mousedown 先清空 `editingCell`，导致 textarea blur 触发的 `commitEdit` 被跳过，输入内容丢失。

**修复**：在所有 mousedown 处理函数（onCellMouseDown/onRowHeaderMouseDown/onColHeaderMouseDown）开头，若正在编辑则先调用 `commitEdit()` 保存内容，再执行选区切换。

### 7.14 插入图片（Image）

- 某些历史模板中，同列已存在 `down` 主格后又残留了无主格 `right` 根，预览时会把旁侧空白单元格错误填充（如 B8/C8）。
- 设计器中无法直观看出“数据集变量单元格”当前是向下还是向右展开，容易误配。

**方案**：
- `ExpandEngine` 在处理根 `right` 展开时增加“疑似重复根”识别：
  - 条件：`col=0`、无 `leftMasterCell/topMasterCell`、有 `dataset/fieldName`，且同列上方存在同 `dataset.field` 的 `down` 根。
  - 行为：跳过该 `right` 根并清理其渲染残留内容，同时输出结构化告警（非覆盖冲突告警）。
- `CellCanvas` 在设计器单元格右上角新增展开方向徽标：
  - `↓` 表示数据集变量向下展开；
  - `→` 表示数据集变量向右展开；
  - 仅对主单元格且已绑定 `dataset.field` 的展开单元格显示。
- 回归测试：
  - `test/attachment-template-regression.test.ts` 增加 v2 附件场景断言（B8/C8 不再出现错误填充）。
  - `test/expand-layout-regressions.test.ts` 同步覆盖重复右根抑制语义。

### 7.38 v3 预览重复右展开修复 + 指示图标紧凑化 + 新预制示例模板

| 维度 | UReport2 | VReport | 原因 |
| --- | --- | --- | --- |
| 运行环境 | Java 后端 + 浏览器 | 纯浏览器 | 项目定位 |

| HTTP API 数据源 | ❌ 无 | ⏳ v0.2 | v0.1 用 Excel 上传替代 |
| Excel/CSV/JSON 数据源 | ❌ 无 | ✅ | 适配前端 |
| Spring Bean 数据源 | ✅ | ❌ 不支持 | 无后端 |
| 内置函数 | 100+ | 40+（常用） | 范围裁剪 |
| 导出 PDF/Excel/Word/HTML | ✅ | ⏳ v0.2 | v0.1 聚焦设计与预览 |
| 子报表 | ✅ | ⏳ v0.2 | 范围裁剪 |
| 二维码/条形码 | ✅ | ✅ | 已实现 |
| 图片 | ✅ | ✅ | URL + base64 |
| 斜线表头 | ✅ | ⏳ v0.2 | 范围裁剪 |
| 模板存储 | 服务器文件系统 | IndexedDB + 文件 | 纯前端 |
| 模板格式 | XML | JSON | 前端友好 |
| 表达式语法 | 基本对齐 | 基本对齐 | 降低迁移成本 |
| 单元格迭代模型 | ✅ | ✅ 完整实现 | 核心能力 |
