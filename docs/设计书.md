## 1. 项目概述

### 1.1 背景与目标

UReport2 是一款基于 Spring 架构的纯 Java 高性能报表引擎，通过**单元格迭代模型**可实现任意复杂的中国式报表，并提供基于网页的可视化报表设计器。其核心能力包括：单元格迭代展开、表达式引擎、多数据源、图表、交叉表、参数面板、条件格式、多格式导出等。

UReport2 强依赖 Java 后端（JDBC 直连数据库、Spring Bean 数据源、服务器端渲染与导出），无法在纯前端环境运行。**VReport 的目标是用纯前端技术栈（Vue3 + TypeScript）复刻 UReport2 的核心功能**，让用户在浏览器中即可完成报表的设计与预览，无需部署任何后端服务。

### 1.2 设计原则

1. **零后端依赖**：所有计算、渲染均在浏览器端完成。
2. **模型对齐 UReport**：单元格模型、迭代语义、表达式语法尽量与 UReport2 保持一致，降低用户迁移成本。
3. **数据源适配前端**：用 JSON / CSV / Excel 文件上传 替代 JDBC 直连，覆盖 UReport 的数据集场景（v0.1 不含 HTTP API）。
4. **模板可迁移**：报表模板以 JSON 描述，支持 IndexedDB 持久化与文件导入导出。
5. **渐进式实现**：核心模块优先，导出与高级特性作为扩展预留。

## 4. 核心模块设计

### 4.1 单元格模型（Cell Model）

UReport 的核心是**单元格**。每个单元格有名称（如 A1）、坐标、合并跨度、内容、展开方向、主格关系等属性。

#### 4.1.1 数据结构

```typescript
// 单元格类型
type CellType = 'text' | 'image' | 'chart' | 'formula' | 'qrcode' | 'barcode';
// 展开方向
type ExpandDirection = 'none' | 'down' | 'right';
// 聚合方式
type Aggregate = 'none' | 'sum' | 'avg' | 'count' | 'max' | 'min' | 'group' | 'distinct';

interface Cell {
  id: string;
  name: string;              // 显示名 A1, B2
  row: number;               // 行索引（0-based）
  col: number;               // 列索引（0-based）
  rowSpan: number;           // 行合并数
  colSpan: number;           // 列合并数
  content: string;           // 原始内容（文本或表达式）
  cellType: CellType;
  expandDirection: ExpandDirection;
  // 主格关系（控制迭代分组）
  leftMasterCell?: string;   // 左主格名，控制向下展开时的分组
  topMasterCell?: string;    // 上主格名，控制向右展开时的分组
  // 数据绑定
  dataset?: string;          // 绑定的数据集名
  fieldName?: string;        // 绑定字段名
  aggregate?: Aggregate;     // 聚合方式
  // 样式与格式
  styleId?: string;          // 引用 styles 中的样式
  numberFormat?: string;     // 数字格式 "#,##0.00"
  dateFormat?: string;       // 日期格式 "yyyy-MM-dd"
  // 条件格式
  conditionIds?: string[];   // 引用 conditionFormats
  // 图表/图片配置
  chartConfig?: ChartConfig;
  imageConfig?: ImageConfig;
  // 其他
  remark?: string;           // 备注
  printable?: boolean;       // 是否参与打印
}

interface RowConfig {
  index: number;
  height: number;            // px
  hidden?: boolean;
  pageBreakBefore?: boolean; // 分页符
  repeat?: boolean;          // 重复表头
}

interface ColumnConfig {
  index: number;
  width: number;             // px
  hidden?: boolean;
  freeze?: boolean;          // 冻结列
}
```

#### 4.1.2 网格结构

设计期使用二维数组 `Cell[][]` 存储原始模板网格（合并单元格用 `rowSpan/colSpan` 标记，被合并的位置存 `null` 或引用主单元格）。

渲染期由迭代引擎生成 `RenderedCell[][]`，其行/列数可能远大于模板（因展开）。

### 4.2 迭代引擎（Expand Engine）⭐核心

UReport 区别于普通表格的关键：**单元格可绑定数据集并按方向展开**，展开过程中通过"主格"关系维持分组层级。

#### 4.2.1 展开语义

- **向下展开（expandDirection = 'down'）**：单元格绑定数据集字段后，按数据集行数纵向复制该单元格所在行，每行绑定一条数据。
- **向右展开（expandDirection = 'right'）**：按数据集行数横向复制该单元格所在列。
- **主格（Master Cell）**：当一个单元格 A1 向下展开为多行时，其右侧的 B1 默认也跟随 A1 展开并绑定同一行数据；若 B1 也设置向下展开，则 B1 在 A1 的每一行内再展开。这种父子关系通过 `leftMasterCell` / `topMasterCell` 显式声明。

#### 4.2.2 算法流程

```
1. 扫描所有单元格，构建主格关系树（MasterTree）
   - 根节点：无主格的展开单元格
   - 子节点：以父格为主格的展开单元格
2. 对每个根节点执行展开：
   expand(cell, parentContext):
     data = cell.dataset ? dataset[cell.dataset] : [{}]
     for each row in data:
       ctx = parentContext.derive(row, cell)
       复制 cell 所在行/列到结果网格
       递归 expand(子节点, ctx)
3. 处理非展开单元格：按主格上下文填充
4. 计算聚合单元格（sum/avg/count...）基于其子主格范围
5. 输出 RenderedCell[][]
```

#### 4.2.3 上下文（Context）

```typescript
interface ExpandContext {
  // 当前单元格所在迭代层的数据行
  rowData: Record<string, any>;
  datasetName: string;
  // 父级上下文链（用于表达式解析时回溯）
  parent?: ExpandContext;
  // 当前展开后在结果网格中的坐标范围
  rowRange: [number, number];
  colRange: [number, number];
}
```

表达式求值时，`${field}` 从当前上下文 `rowData` 取值；若当前层无该字段，沿 `parent` 链向上查找（实现主子报表的数据继承）。

### 4.3 表达式引擎（Expression Engine）

#### 4.3.1 语法设计（对齐 UReport）

| 语法 | 含义 | 示例 |
| --- | --- | --- |
| `${expr}` | 简单表达式（字段引用或计算） | `${name}`、`${price * count}` |
| `=expr` | 公式表达式（支持函数） | `=sum(A1)`、`=if(${age}>18,"成年","未成年")` |
| `A1` | 单元格引用（相对） | `=A1+B1` |
| `$A1` | 绝对列引用 | 展开后仍指向 A 列 |
| `A$1` | 绝对行引用 | 展开后仍指向第 1 行 |
| `$A$1` | 完全绝对引用 | |
| `&` | 字符串拼接 | `=${name}&"-"&${age}` |

#### 4.3.2 模块结构

```
core/expression/
├── Lexer.ts          # 词法分析：token 流
├── Parser.ts         # 语法分析：AST
├── Evaluator.ts      # 求值器：AST + Context → 值
├── functions.ts      # 内置函数注册表
├── operators.ts      # 运算符定义
└── types.ts
```

#### 4.3.3 内置函数（v0.1 约 40+）

分类：数学（sum/avg/count/max/min/round/abs...）、字符串（concat/substring/upper/lower/replace...）、日期（now/year/month/day/formatDate...）、逻辑（if/and/or/not/case...）、聚合（sum/avg/count/max/min/group...）、格式化（format/numberFormat...）。

### 4.4 数据源（Data Source）

#### 4.4.1 统一接口

```typescript
interface DataSource {
  id: string;
  name: string;
  type: 'json' | 'csv' | 'excel' | 'file';
  config: JsonConfig | CsvConfig | ExcelConfig | FileConfig;

interface DataSet {
  id: string;
  name: string;              // 数据集名，供单元格引用
  sourceId: string;          // 所属数据源

  extractor: {
```

#### 4.4.2 四种 Provider

| Provider | 配置 | 取数方式 |
| --- | --- | --- |
| `JsonProvider` | rawJson, dataPath | 解析粘贴的 JSON，按路径取数组 |
| `CsvProvider` | rawCsv, delimiter, hasHeader | PapaParse 解析为数组 |
| `ExcelProvider` | file(上传.xlsx), sheet, headerRow | SheetJS 解析指定 sheet 为对象数组 |
| `FileProvider` | file(上传), format(json/csv/xlsx) | 读取文件内容后委托给对应解析器 |

> **说明**：v0.1 不含 HTTP API 数据源。需要连接后端数据库的场景，请先用 Excel 导出数据再上传，或等待 v0.2 的 HTTP API 支持。

### 4.10 图片（Image）

```typescript
interface ImageConfig {
  source: 'url' | 'base64';     // 来源:URL 引用 或 本地上传(base64 嵌入)
  url?: string;                 // URL 模式,支持 ${param.xxx} 表达式
  base64?: string;              // base64 数据(不含 data: 前缀)
  mimeType?: string;            // image/png 等
  width?: number;               // 0=自适应单元格
  height?: number;
  fit?: 'fill' | 'contain' | 'cover' | 'none';  // 对应 object-fit
}
```

- **URL 模式**：`content` 字段存储 URL 表达式（如 `https://example.com/${param.logo}.png`），运行时由表达式引擎求值为实际图片地址。
- **base64 模式**：图片二进制转 base64 字符串嵌入模板，`content` 设为 `[image:base64]` 占位。无网络依赖，但会增加模板体积（>2MB 建议改用 URL）。
- 渲染期由 [RenderedTable.vue](file:///d:/AI/trae/VReport/src/components/render/RenderedTable.vue) 输出 `<img>`，base64 模式直接拼 `data:${mimeType};base64,${base64}`，URL 模式使用 `cell.value`（已求值）。

- `content` 字段存储原始数据表达式，运行时由表达式引擎解析后传入 `QRPreview.resolvedData`。
- 设计期预览把 `${...}` 表达式替换为占位文本 `SAMPLE`，便于用户直观看到二维码形态。

## 7. 关键技术难点与方案

### 7.1 单元格迭代展开（最高难度）

**难点**：UReport 的主格关系可形成多层嵌套，展开后行/列数动态变化，且表达式需在正确上下文中求值。

**方案**：
- 设计期显式声明 `leftMasterCell/topMasterCell`，避免隐式推断的歧义。
- 展开算法分两阶段：①构建主格树；②深度优先递归展开，每个展开层维护独立 `ExpandContext`。
- 表达式求值时，字段引用沿上下文链向上查找；单元格引用按"展开后坐标"解析。
- 提供可视化主格关系视图，降低用户配置难度。

### 7.2 表达式引擎

**难点**：需同时支持字段引用、单元格引用（含绝对/相对）、函数调用、运算符优先级，且要在迭代上下文中正确求值。

**方案**：自研递归下降解析器，生成 AST 后求值。单元格引用在求值阶段通过 `CellRefResolver` 查询展开后网格，支持绝对/相对语义。

### 7.3 大数据量渲染性能

**难点**：数据集行数过多时，展开后网格可能数万行，DOM 渲染卡顿。

**方案**：
- 预览默认仅渲染前 500 行，提示用户分页或导出查看完整数据。
- 渲染采用虚拟滚动（vue-virtual-scroller 思路）。
- 导出时不走 DOM，直接从 `RenderedCell[][]` 流式写入（ExcelJS 支持流式）。

### 7.4 PDF 导出分页

**难点**：jsPDF + html2canvas 截图方式难以精确按纸张分页，长表格易被截断。

**方案**：
- 渲染期由 `Paginator` 按纸张大小与行高预先计算分页点，在 `RenderedCell[][]` 上标记分页边界。
- 导出时按页生成 canvas，逐页 addImage。表头行按 `page.headerRows` 在每页重复。

### 7.5 HTTP 数据源跨域

**难点**：浏览器同源策略限制直接 fetch 第三方接口。

**方案**：
- 文档明确要求目标接口开启 CORS。
- 提供可选代理配置：用户在设置中填写代理 URL 前缀，请求时改走代理。
- 不内置后端代理服务（保持纯前端）。

### 7.6 撤销/重做

**难点**：设计器需支持完整的操作历史。

**方案**：采用命令模式（Command Pattern），每个设计操作封装为可逆 Command，压入历史栈。Pinia 中维护 `undoStack/redoStack`。

### 7.7 Excel 风格设计器交互

**难点**：用户期望报表设计器的交互与 Excel 一致，包括直接输入、多选、快捷键、拖拽选区等。

**方案**：
- **键盘直接编辑**：选中单元格后按任意可打印字符即进入编辑模式并输入该字符；F2 进入编辑保留原内容；Enter 确认并下移；Tab 确认并右移。
- **拖拽多选**：鼠标按下后拖动，通过 `getCellFromPoint` 实时计算行列索引并扩展选区。
- **方向键导航**：上下左右键移动选中单元格；Shift+方向键扩展选区。
- **Delete/Backspace**：清除选区内所有单元格内容。
- **Ctrl+B/I/U**：切换加粗/斜体/下划线，支持批量应用到多选区域。
- **样式工具栏（CellToolbar）**：画布上端提供图标按钮区，包含字体/字号/加粗/斜体/下划线/字体颜色/背景色/对齐/边框/合并拆分/插入行列，所有操作支持多选批量应用。
- **Grid.applyStyleToRange / setBorderToRange**：网格层提供批量样式与边框应用方法，支持对任意矩形区域一次性设置。

### 7.8 数据源与数据集编辑

**难点**：初版仅支持新建和删除，用户修改配置需删除重建。

**方案**：DataPanel 增加编辑功能，复用新建对话框，根据 `editingDs/editingSet` 状态区分新建与更新。编辑数据源后自动清除关联数据集缓存，确保下次取数使用新配置。

### 7.9 预制报表参数示例

**方案**：两个预制模板均内置参数示例：
- 销售明细报表：`region`（下拉单选：全部/华东/华南/华北/西南）、`minAmount`（数字：最低销售额）
- 学生成绩统计表：`class`（下拉单选：全部/一班/二班）、`subject`（下拉单选：全部/语文/数学）、`passScore`（数字：及格线）

参数在预览时通过 `${param.参数名}` 在表达式中引用，触发数据集重新取数与报表重渲染。

### 7.10 公式输入联想（Excel 风格）

**难点**：用户输入公式时需要参考 Excel 的联想提示，降低记忆成本。

**方案**：
- 新增 [Autocomplete.ts](file:///d:/AI/trae/VReport/src/core/expression/Autocomplete.ts) 模块，提供 `matchSuggestions`（函数名前缀匹配）和 `getParamHint`（括号内参数签名提示）。
- CellCanvas 编辑器集成联想下拉列表：输入 `=` 开头后，键入函数名前缀自动弹出匹配列表（最多 8 项），上下键导航，Enter/点击选中后自动补全函数名和括号。
- 参数提示框：光标进入括号内时，底部显示当前函数签名和说明。

### 7.11 预览默认展示

**方案**：进入预览页时自动用参数默认值渲染报表。参数默认值留空表示"全部"，即展示所有数据行，无需手动点击查询。

### 7.12 面板可收起

**方案**：设计器左右面板支持点击收起/展开，收起后仅保留 20px 折叠条，最大化画布空间。右侧面板折叠条在左侧（order: -1），左侧面板折叠条在右侧。

### 7.13 单元格编辑提交修复

**问题**：点击其他单元格时，mousedown 先清空 `editingCell`，导致 textarea blur 触发的 `commitEdit` 被跳过，输入内容丢失。

**修复**：在所有 mousedown 处理函数（onCellMouseDown/onRowHeaderMouseDown/onColHeaderMouseDown）开头，若正在编辑则先调用 `commitEdit()` 保存内容，再执行选区切换。

### 7.14 插入图片（Image）

**难点**：图片需要支持本地上传和 URL 引用两种方式；URL 模式要支持 `${param.xxx}` 表达式动态替换；base64 模式不能过度膨胀模板体积。

**方案**：
- `ImageConfig.source` 区分 `url` / `base64` 两种来源。
- [ImageDialog.vue](file:///d:/AI/trae/VReport/src/components/designer/ImageDialog.vue) 提供单选切换：URL 模式用 `a-input` 输入；本地上传用 `a-upload` 的 `before-upload` 拦截，通过 `FileReader.readAsDataURL` 转 base64，>2MB 时给出建议提示。
- URL 模式的 `content` 存储原始 URL 表达式，运行时由表达式引擎求值得到实际图片地址。
- 渲染期由 `cellStyleOf` + `<img>` 配合 `object-fit` 实现缩放模式（contain/cover/fill/none）。

### 7.15 插入二维码（QRCode）

**难点**：二维码数据需要支持静态文本和表达式；设计期无法实际求值表达式，需给出直观预览。

**方案**：
- 引入 [qrcode](https://www.npmjs.com/package/qrcode) 库，渲染到 `<canvas>`。
- [QRCodeDialog.vue](file:///d:/AI/trae/VReport/src/components/designer/QRCodeDialog.vue) 提供数据输入、纠错等级（L/M/Q/H）、像素大小、边距、前景/背景色配置，并提供 160px 实时预览。
- 设计期预览组件 [QRPreview.vue](file:///d:/AI/trae/VReport/src/components/designer/QRPreview.vue) 把 `${...}` 表达式替换为 `SAMPLE` 占位文本，避免二维码渲染失败。
- 运行期 `resolvedData` prop 由 `RenderedTable` 传入 `cell.value`（表达式引擎已求值），覆盖配置中的原始表达式。

### 7.16 插入条码（Barcode）

**难点**：条码格式多样（CODE128/EAN13/UPC 等），每种格式对数据有严格校验规则；数据不符需友好提示。

**方案**：
- 引入 [jsbarcode](https://github.com/lindell/JsBarcode) 库，渲染到 `<svg>`。
- [BarcodeDialog.vue](file:///d:/AI/trae/VReport/src/components/designer/BarcodeDialog.vue) 支持 9 种格式，配置宽度/高度/显示文本/前景背景色。

- 设计期预览把 `${...}` 替换为 `1234567890` 占位数据，保证条码形态可见。

### 7.17 插入图表（Chart）

**难点**：图表配置项复杂（类型/数据集/分类字段/多系列）；设计期需要实时预览；运行期需从数据集缓存取数。

### 7.21 主格自动计算与画布拾取

**问题**：左右主格（`leftMasterCell`/`topMasterCell`）原本需手动输入单元格名，用户易写错且不理解主格语义。

**方案**：
- [PropertyPanel.vue](file:///d:/AI/trae/VReport/src/components/designer/PropertyPanel.vue#L172-L203) 主格输入框默认**只读**，显示自动计算结果；右侧新增「自定义选择」按钮（`AimOutlined`），点击进入拾取模式后输入框可编辑、按钮高亮。
- **自动计算算法**（`autoCalcMaster(which)`）：
  - 左主格：当前单元格 `expandDirection === 'down'` 时，向左扫描同行（`col` 从 `cur.col-1` 到 `0`），找到第一个 `expandDirection === 'down'` 的单元格，返回其名称（`colIndexToName(cell.col) + (cell.row+1)`）。
  - 上主格：当前单元格 `expandDirection === 'right'` 时，向上扫描同列（`row` 从 `cur.row-1` 到 `0`），找到第一个 `expandDirection === 'right'` 的单元格，返回其名称。
  - 未找到则返回空字符串，输入框 placeholder 显示「自动」。
- **手动拾取流程**：
  - [designer.ts](file:///d:/AI/trae/VReport/src/stores/designer.ts#L26-L28) 新增 `masterPicking: ref<'left'|'top'|null>(null)` 与 `pickedMasterCell: ref<string|null>(null)`。
  - PropertyPanel 点击拾取按钮时设置 `designer.masterPicking = 'left'|'top'`。
  - [CellCanvas.vue](file:///d:/AI/trae/VReport/src/components/designer/CellCanvas.vue) 的 `onCellMouseDown` 优先检测 `designer.masterPicking`：若处于拾取模式，填充 `designer.pickedMasterCell = colIndexToName(col) + (row+1)` 并退出拾取模式，不触发选区切换。
  - PropertyPanel watch `designer.pickedMasterCell`，将值写入对应主格字段并同步到输入框。
- 切换展开方向（`onExpandChange`）时重置手动模式并重新自动计算。

### 7.25 可变参数聚合函数（Excel 风格）

**方案**：
- [functions.ts](file:///d:/AI/trae/VReport/src/core/expression/functions.ts) `sum`/`avg`/`count`/`max`/`min`/`distinct` 由 `argCount: 1` 改为 `argCount: -1`（可变参），`a.flatMap(flatten)` 展平所有参数后聚合。
- 兼容 Excel 风格三种写法：`sum(A1,A2)`、`sum(A1:A3)`、`sum(A1+A2)`。

### 7.26 公式示例提示与格式校验（Excel 风格）

- [CellCanvas.vue](file:///d:/AI/trae/VReport/src/components/designer/CellCanvas.vue#L124-L132) 参数提示区分两行：首行签名+描述，次行绿色等宽字体显示示例；编辑时实时调用 `validateFormula`，错误以红色提示框显示在编辑器下方。

### 7.27 公式引用单元格彩色高亮（Excel 风格）

- [ExpandEngine.ts](file:///d:/AI/trae/VReport/src/core/engine/ExpandEngine.ts#L240-L251) `fillRowCells` 不再用 `source.row === masterCell.row` 比较（粘贴后主格行号已变），改为填充同行所有未设置 context 的单元格，确保从属格跟随主格展开。

### 7.30 参数过滤非破坏性还原

**问题**：[ParameterEngine.ts](file:///d:/AI/trae/VReport/src/core/parameter/ParameterEngine.ts) 的 `applyToDataSets` 直接覆写 `cachedRows`，再次查询时基于已过滤数据再过滤，数据越来越少直至为空。

**方案**：
- [DataSet](file:///d:/AI/trae/VReport/src/types/index.ts#L57-L58) 新增 `originalRows` 字段备份原始取数结果。
- `applyToDataSets` 始终从 `originalRows` 过滤；[PreviewView.render()](file:///d:/AI/trae/VReport/src/views/PreviewView.vue#L118-L124) 重新取数后重置 `originalRows`。

### 7.31 公式聚合器递归求值与循环引用提示

**问题**：[Aggregator.ts](file:///d:/AI/trae/VReport/src/core/engine/Aggregator.ts) 使用固定 5 轮迭代求值，超过 5 层的单元格依赖链无法收敛；循环引用静默返回 undefined，用户无法感知。

**方案**：
- `evaluateAll` 改为按需递归：遍历所有单元格调用 `getCellValue`，遇到依赖自动递归求值，无层数限制。
- 循环引用检测返回 `#CIRC!` 提示字符串，而非静默 undefined。
- 所有求值结果（含错误）统一缓存，避免重复计算。

### 7.32 图表单元格响应式与内存泄漏修复

**问题**：[ChartCell.vue](file:///d:/AI/trae/VReport/src/components/render/ChartCell.vue) 的 width/height 使用 `const` 初始化，props 变化后尺寸不更新；`watch(props.config)` 未在更新前 dispose 旧 ECharts 实例，频繁刷新可能导致内存泄漏。

**方案**：
- width/height 改为 `computed`，根据 props 实时计算。
- `style` 改用 `computed` 绑定到 DOM。
- `watch(props.config)` 回调中更新前先 `chart.dispose()` 并置空，再重新 `echarts.init` 渲染。
- 新增 `watch([width, height], ...)` 调用 `chart.resize()`，实现尺寸自适应。

### 7.33 IndexedDB 保存深拷贝保留 Date/Map/Set

**问题**：[db.ts](file:///d:/AI/trae/VReport/src/utils/db.ts) 使用 `JSON.parse(JSON.stringify(template))` 剥离 Proxy，会丢失 `Date` 对象（转为字符串）、`Map`、`Set`、`undefined` 等。

**方案**：
- 新增 `deepClone` 函数，优先使用 `structuredClone`（现代浏览器原生支持，可正确保留 Date/Map/Set）。
- 不可用时回退 `customDeepClone`：显式处理 Date、Map、Set，跳过函数，使用 WeakMap 防循环引用。
- `saveTemplate` 调用 `deepClone(template)` 替代 JSON 深拷贝。

### 7.34 条件格式单元格名大小写兼容

**问题**：[ConditionEngine.ts](file:///d:/AI/trae/VReport/src/core/format/ConditionEngine.ts) 的 `compareCellName` 正则仅支持大写字母，用户输入小写范围（如 `a1:a10`）时无法匹配。

**方案**：正则放宽为 `[A-Za-z]+`，比较前统一转大写。

### 7.35 日期参数统一格式化

**问题**：[ParameterForm.vue](file:///d:/AI/trae/VReport/src/components/preview/ParameterForm.vue) 中 `a-date-picker` 返回 Dayjs 对象，而 [ParameterEngine.ts](file:///d:/AI/trae/VReport/src/core/parameter/ParameterEngine.ts) 使用 `String(cellVal) === String(val)` 比较；Dayjs 的字符串表示与数据集中常见的 `YYYY-MM-DD` 可能不一致，导致日期过滤失效。

**方案**：在 `ParameterForm.vue` 提交查询时，将 `date` 类型参数统一格式化为 `YYYY-MM-DD` 字符串后再 emit。
