## 1. 快速开始

### 1.1 环境要求

- Node.js ≥ 18
- 现代浏览器：Chrome / Firefox / Edge（不支持 IE）

### 1.2 安装与启动

```bash
# 安装依赖
npm install

# 开发模式
npm run dev

# 构建生产包
npm run build
```

启动后浏览器访问 `http://localhost:5173`，默认进入模板列表页。

### 1.3 三分钟做出第一张报表

1. 在模板列表页点击「新建报表」。
2. 进入设计器，左侧「数据源」面板 → 点击「+」→ 选择「JSON 数据」→ 粘贴示例数据 → 保存。
3. 左侧「数据集」面板 → 「+」→ 选择刚建的数据源 → 命名 `ds1` → 保存。
4. 在画布 A1 单元格输入 `${ds1.name}`，B1 输入 `${ds1.score}`。
5. 选中 A1，右侧属性面板「展开方向」选「向下」。
6. 点击顶部「预览」→ 看到数据按行展开。
7. 点击「导出」→ 选择 Excel → 下载文件。

## 3. 数据源配置

VReport 支持四种数据源，均在前端完成取数。v0.1 不含 HTTP API 数据源（计划 v0.2 支持）。

### 3.1 Excel 上传数据源（推荐）

适用于已有结构化数据的场景。

**配置步骤**：
1. 左侧「数据源」Tab → 「+」→ 选「Excel 上传」。
2. 上传 `.xlsx` 文件（支持多 sheet）。
3. 选择目标 sheet 名（默认第一个）。
4. 配置首行是否为表头（默认是）。
5. 点击「预览」→ 查看前 10 条解析结果。
6. 「保存」。

> 解析后的数据会随模板一起存入 IndexedDB，大文件会占用浏览器存储空间。

### 3.2 JSON 数据源

适用于快速演示或静态数据。

1. 「+」→ 选「JSON 数据」。
2. 在文本框粘贴 JSON，例如：
   ```json
   [
     {"name":"张三","score":85},
     {"name":"李四","score":92}
   ]
   ```
3. 「数据路径」留空（直接是数组）或填数组所在路径（如 `data.list`）。
4. 保存。

### 3.3 CSV 数据源

1. 「+」→ 选「CSV 数据」。
2. 粘贴 CSV 文本或上传 `.csv` 文件。
3. 配置：分隔符（默认逗号）、是否首行表头。
4. 保存。

### 3.4 通用文件上传数据源

1. 「+」→ 选「文件上传」。
2. 上传 `.json` / `.csv` / `.xlsx` 文件。
3. 系统根据扩展名自动选择解析器并预览。
4. 保存。

> 文件内容会随模板一起存入 IndexedDB，注意大文件会占用浏览器存储空间。

## 4. 数据集管理

数据源是"连接"，数据集是"取数结果"。一个数据源可派生多个数据集。

### 4.1 新建数据集

1. 左侧「数据集」Tab → 「+」。
2. 填写：
   - **名称**：数据集名（如 `ds1`），表达式中通过 `${ds1.字段名}` 引用。
   - **数据源**：选择已建数据源。
   - **取数路径/sheet**：覆盖数据源默认路径或 sheet。
   - **参数过滤**：可对数据集行按参数 `${param.xxx}` 做前端过滤（v0.1 不支持参数化取数，仅本地过滤）。
3. 「预览数据」→ 确认字段。
4. 「字段映射」：可重命名字段、转换类型（string/number/date）。
5. 保存。

### 4.2 数据集刷新

设计期点击数据集「刷新」按钮可重新取数并缓存，供画布预览使用。

## 6. 迭代展开（核心）

迭代展开是 VReport 区别于普通表格的核心能力，用于将数据集按行/列展开。

### 6.1 向下展开

**场景**：将数据集每条记录展开为一行。

1. 选中绑定数据集字段的单元格（如 A1 内容 `${ds1.name}`）。
2. 属性面板「展开方向」→ 选「向下」。
3. 预览：A1 所在行被复制 N 份（N = 数据集行数），每份绑定一条记录。

### 6.2 向右展开

**场景**：将数据按列展开（如月份横向排列）。

1. 选中单元格。
2. 「展开方向」→ 选「向右」。
3. 预览：单元格所在列被复制 N 份。

### 6.3 主格关系（分组）

**场景**：分类汇总。如按"部门"分组，每个部门下展开多个"员工"。

1. A1 输入 `${ds1.dept}`，展开方向「向下」，**无主格**（根展开）。
2. B1 输入 `${ds1.name}`，展开方向「向下」，**左主格**填 `A1`。
3. 预览：A1 按部门去重展开，B1 在每个部门内展开该部门员工。

> **左主格**：控制向下展开时的分组父级。
> **上主格**：控制向右展开时的分组父级。

### 6.4 聚合单元格

在分组场景下，对子主格范围求聚合：

1. C1 输入 `${ds1.salary}`，左主格 `A1`，展开方向「向下」。
2. D1 输入 `=sum(C1)`，左主格 `A1`，**展开方向「无」**，聚合方式「sum」。
3. 预览：D1 显示每个部门的薪资合计。

支持的聚合：sum / avg / count / max / min / distinct / group。

## 7. 表达式与函数

### 7.1 表达式语法

| 写法 | 含义 |
| --- | --- |
| `${ds1.name}` | 引用数据集字段 |
| `${param.deptId}` | 引用报表参数 |
| `=A1+B1` | 单元格引用相加 |
| `=sum(A1:A10)` | 区域聚合 |
| `=if(${age}>18,"成年","未成年")` | 条件判断 |
| `${name}&"-"&${age}` | 字符串拼接 |
| `$A1` / `A$1` / `$A$1` | 绝对引用 |

### 7.2 单元格引用规则

- **相对引用** `A1`：随当前单元格位置相对偏移。
- **绝对列** `$A1`：列固定为 A，行相对。
- **绝对行** `A$1`：行固定为 1，列相对。
- **完全绝对** `$A$1`：始终指向展开后网格的 A1 位置。

### 7.3 公式编辑器

点击属性面板「公式」右侧的 `fx` 按钮打开公式编辑器：
- 左侧函数分类树
- 中间编辑区（带语法高亮）
- 右侧参数说明与示例
- 底部「校验」按钮检查语法

## 9. 交叉表

交叉表是行列表头交叉汇总的报表结构。

### 9.1 手动构建交叉表

1. 行表头：A1 绑定 `${ds1.region}`，向下展开，无主格。
2. 列表头：B1 绑定 `${ds1.month}`，向右展开，上主格 `A1`。
3. 数据格：B2 绑定 `${ds1.amount}`，左主格 `A1`、上主格 `B1`，聚合 sum。
4. 预览：得到地区 × 月份的交叉汇总表。

### 9.2 交叉表向导

工具栏「插入」→「交叉表向导」：
1. 选择数据集。
2. 指定行字段、列字段、值字段、聚合方式。
3. 自动生成交叉表结构并插入画布。

## 10. 参数面板

参数用于在预览时动态控制数据查询。

### 10.1 新建参数

左侧「参数」Tab → 「+」：
- **参数名**：`deptId`（表达式中用 `${param.deptId}`）
- **标签**：显示文字「部门」
- **类型**：string / number / date / datetime / select / multiSelect / cascader
- **默认值**：预览初始值
- **必填**：是否强制输入
- **下拉选项**：手填或绑定数据集字段
- **级联父参数**：选择父参数后，本参数选项随父参数变化

### 10.2 参数绑定到数据集

在数据集的「参数过滤」中，可按参数 `${param.deptId}` 对数据集行做前端过滤。预览时改变参数值 → 触发数据集重新过滤 → 报表重渲染。

> v0.1 数据源为本地文件（Excel/JSON/CSV），参数仅支持本地过滤；v0.2 引入 HTTP API 后支持参数化取数。

### 10.3 预览时填参

进入预览页，顶部显示参数表单，填值后点击「查询」即刷新报表。

## 11. 条件格式

根据单元格值或表达式动态改变样式。

### 11.1 新建条件格式

属性面板「条件格式」→ 「+」：
- **名称**：规则名
- **应用范围**：`A1:A10` 或单元格名
- **规则类型**：
  - **单元格值**：值 > 90 时高亮
  - **公式**：自定义表达式 `=${score}>90`
  - **数据条**：按值显示长度条
  - **图标集**：按值显示图标
  - **前 N 项**：高亮最大/最小 N 项
- **命中样式**：字体颜色、背景色、加粗等

### 11.2 多规则优先级

同一范围可加多条规则，按列表顺序从上到下匹配，先命中先生效（可在属性面板拖拽调整顺序）。

## 14. 导出报表 ⏳ v0.2

> **v0.1 暂不支持导出功能。** 以下为 v0.2 规划，届时将在预览页右上角提供「导出」下拉。

| 格式 | 说明 |
| --- | --- |
| Excel (.xlsx) | 保留单元格值、合并、样式、列宽行高、图表（转为图片） |
| PDF (.pdf) | 按页面配置分页，表头每页重复，图表以图片嵌入 |

## 15. 模板管理

### 15.1 模板列表

首页为模板列表，展示所有本地模板（存于 IndexedDB）：
- 新建 / 复制 / 删除 / 重命名
- 搜索 / 按标签筛选
- 显示最后修改时间

### 15.2 保存

- 设计器中 `Ctrl+S` 或工具栏「保存」。
- 首次保存需填写名称、描述、标签。

### 15.3 导入导出模板

- **导出**：模板列表 → 模板卡片 → 「导出」→ 下载 `.vreport.json` 文件。
- **导入**：模板列表 → 「导入」→ 选择 `.vreport.json` 文件。
- 文件包含完整模板定义（含数据源配置，但不含 HTTP 数据源的真实数据，预览时实时取数）。

### 15.4 存储说明

- 模板存于浏览器 IndexedDB，不同浏览器/设备不共享。
- 跨设备迁移请用「导出文件 → 导入文件」。
- 清除浏览器数据会丢失模板，请定期导出备份。

## 17. 常见问题

### Q1：能否连接 MySQL/Oracle 数据库？
A：v0.1 纯前端无法直连数据库。请先用数据库工具导出 Excel/CSV，再上传到 VReport 作为数据源。v0.2 计划支持 HTTP API 数据源，届时可通过后端接口取数。

### Q2：预览时数据不展开？
A：检查：① 单元格是否绑定了数据集字段；② 展开方向是否设置；③ 数据集是否有数据（点数据集「预览」确认）。

### Q3：表达式 `${ds1.name}` 显示原样未解析？
A：确认数据集名拼写正确；确认单元格所在行/列处于该数据集的展开上下文中（即有同数据集的展开单元格作为主格）。

### Q4：v0.1 支持导出吗？
A：v0.1 暂不支持导出功能（PDF/Excel/Word/HTML 均在 v0.2 实现）。如需保存报表结果，可使用浏览器打印（Ctrl+P）另存为 PDF。

### Q5：模板找不到了？
A：模板存于浏览器 IndexedDB。若清除浏览器数据或换设备，需提前用「导出文件」备份。换浏览器后用「导入文件」恢复。

### Q6：大数据量预览卡顿？
A：预览默认仅渲染前 500 行。设计期可在「设置」调低预览行数。

### Q7：能否多人协作编辑同一报表？
A：v0.1 不支持。模板为本地存储，需通过文件导入导出共享。后续版本考虑接入云端存储。

### Q8：与 UReport2 模板能否互通？
A：不能直接互通。UReport2 用 XML 格式且依赖 Java 后端能力，VReport 用 JSON 格式且为纯前端。表达式语法与单元格模型对齐，可手动迁移设计。
